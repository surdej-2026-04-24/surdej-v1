# Changelog — 2026-02

> **Highlights**: AI Chat overhaul — MCP tool integration, tool toggles, markdown table rendering,
> multi-language support, debug panel with full chat export.

---

## Week 9 (Feb 24–25)

### Feature: MCP Tool Toggle Panel
- **Type**: feature
- **Scope**: frontend, api
- **Description**: Added a wrench icon (🔧) in the chat input area that opens a panel showing all 5 MCP tools (RAG Search, Property Search, Property Details, Yield Calculation, Document Listing). Each tool can be toggled on/off individually. The selection is sent with every message so the AI only uses the enabled tools.
- **Files**: `apps/frontend/src/routes/chat/ChatPage.tsx`, `apps/api/src/core/ai/routes.ts`, `apps/api/src/core/ai/chat.ts`

### Feature: Markdown Table Rendering
- **Type**: feature
- **Scope**: frontend
- **Description**: Chat responses now render markdown tables with proper styling — rounded borders, alternating row colors, header highlights, and horizontal scrolling for wide tables. Uses `remark-gfm` plugin for GitHub Flavored Markdown.
- **Files**: `apps/frontend/src/routes/chat/ChatPage.tsx`, `apps/frontend/src/index.css`

### Feature: Multi-Language Response
- **Type**: improvement
- **Scope**: api
- **Description**: Updated the system prompt to detect the user's language and respond accordingly. Previously defaulted to Danish; now responds in English when asked in English, Danish when asked in Danish, etc.
- **Files**: `apps/api/src/core/ai/chat.ts`

### Feature: Chat Debug Panel
- **Type**: feature
- **Scope**: frontend, api
- **Description**: Discrete info icon (ℹ) in the chat header that opens a debug panel showing system prompt, version, AI provider, model tier, conversation ID, and enabled tools. Includes "Copy Full Chat" button that exports the entire conversation with metadata (version, tenant, tools, timestamps, all messages) to clipboard as markdown for analysis.
- **Files**: `apps/frontend/src/routes/chat/ChatPage.tsx`, `apps/api/src/core/ai/routes.ts`
- **API**: `GET /api/ai/debug` endpoint added

### Fix: Tool Calling Reliability
- **Type**: fix
- **Scope**: api
- **Description**: Fixed two issues preventing reliable tool calling: (1) The `onStepFinish` logging handler crashed on `undefined.slice()` when tool args were empty, aborting the entire stream. (2) The old inline RAG retrieval was injecting low-quality document snippets that confused GPT-4o into answering without calling tools. Now RAG retrieval is skipped when tools are enabled.
- **Files**: `apps/api/src/core/ai/chat.ts`

### Feature: Changelog & Release Hub (Instructions)
- **Type**: feature
- **Scope**: docs
- **Description**: Added §17a (Changelog & Feature History) and §17b (Developer Release Hub) sections to the Surdej instructions. Establishes convention for monthly changelogs in `docs/changelog/` and defines a Release Hub page under `/developer/releases`.
- **Files**: `.surdej/agents/guidelines/instructions.md`, `docs/changelog/`
